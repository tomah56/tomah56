## AI & Agent Workflows

A growing focus of my work is building autonomous, AI-driven pipelines — combining API orchestration, agent logic, and real-time processing.

### [Child Safety AI](https://github.com/tomah56/child_safety_ai)
> *Hackathon · 24 hours · Python · FastAPI · OpenAI API · Docker · Chrome Extension*

Led the engineering of an AI-powered content moderation tool built during a child safety hackathon. The system uses OpenAI's moderation API to classify and block harmful text and images in real time, surfaced as a Chrome browser extension. Delivered a production-grade API with Swagger documentation, Docker deployment, and automated test coverage — all within a single day.

**Key decisions:** Chose FastAPI for its async-first design and auto-generated docs; containerised with Docker to enable zero-setup deployment; modular route architecture allows easy integration of additional moderation providers.

## Systems & Backend Engineering

### [Webserv](https://github.com/tomah56/42-webserv)
> *C++ · HTTP/1.1 · Non-blocking I/O · Team of 3*

Co-built an HTTP/1.1 web server in C++ using Nginx as a reference. Handles concurrent connections via non-blocking I/O, supports CGI execution, custom error pages, and chunked transfer encoding. Built with [Alice](https://github.com/akamlah) and [Olawale](https://github.com/Sirelaw).

### [Mini-Shell](https://github.com/tomah56/minishell)
> *C · POSIX · Processes · Pipes*

Implemented a POSIX-compliant shell with builtins, redirections, pipes, subshells, and environment variable expansion. Deep dive into process management and the Unix execution model. Built with Sonja.

### [Cube3D](https://github.com/tomah56/cube3D)
> *C · Ray Casting · Game Engine*

A Wolfenstein-3D–style raycasting engine written in C. Demonstrates real-time rendering fundamentals, trigonometry-based projection, and event-driven input handling. Built with [Olawale](https://github.com/Sirelaw).

### [ft_transcendence](https://github.com/tomah56/ft_transcendence)
> *Full-Stack · Auth · WebSockets · PostgreSQL · Team of 3*

Capstone project of the 42 curriculum. Full-stack multiplayer Pong platform with OAuth authentication, live game rooms, spectator mode, real-time chat, and a user profile/leaderboard system. Built with [doreshev](https://github.com/doreshev) and [pweinstock](https://github.com/pweinstock).

### [Philosopher](https://github.com/tomah56/philosopher_42)
> *C · Multithreading · Mutexes · Race Condition Prevention*

Classic dining philosophers problem — implemented with threads and mutexes, with rigorous attention to deadlock avoidance and resource contention.
